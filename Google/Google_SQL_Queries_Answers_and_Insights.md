# 1. Distinct Search Queries After January 18, 2024

## Question

List all distinct search queries made on or after January 18, 2024, ordered alphabetically.

## SQL Query

```sql
SELECT s.query_text AS query_text
FROM search_queries s
WHERE s.search_date >= '2024-01-18'
GROUP BY s.query_text
ORDER BY s.query_text;
```

## Answer / Result

Returns all unique search queries made on or after January 18, 2024.

## Insight

This query uses **GROUP BY with filtering on date range** to identify unique search behavior in a specific time window, helping analyze recent user interest trends.

# 3. Users Who Never Made a Search Query

## Question

Find users who have never made a search query.

## SQL Query

```sql
SELECT u.user_id AS user_id
FROM users u
LEFT JOIN search_queries s
ON u.user_id = s.user_id
WHERE s.user_id IS NULL
GROUP BY u.user_id
ORDER BY u.user_id;
```

## Answer / Result

Returns users with no search activity.

## Insight

This query uses a **LEFT JOIN with NULL filtering and GROUP BY** to detect inactive users in the search system.

# 11. Users Who Searched on Consecutive Days

## Question

Find users who searched on consecutive days.

## SQL Query

```sql
SELECT s1.user_id AS user_id
FROM search_queries s1
INNER JOIN search_queries s2
ON s1.user_id = s2.user_id
AND s2.search_date = DATE(s1.search_date, '+1 day')
GROUP BY s1.user_id
ORDER BY s1.user_id;
```

## Answer / Result

Returns users with consistent daily engagement.

## Insight

This query uses **self-join logic with GROUP BY** to detect sequential user activity patterns.

# 12. Most Searched Query for Each Day

## Question

Find most searched query per day.

## SQL Query

```sql
SELECT ranked.search_date AS search_date,
ranked.query_text AS query_text
FROM (
SELECT s.search_date AS search_date,
s.query_text AS query_text,
COUNT(*) AS total_searches,
RANK() OVER (
PARTITION BY s.search_date
ORDER BY COUNT(*) DESC
) AS rnk
FROM search_queries s
GROUP BY s.search_date, s.query_text
) ranked
WHERE ranked.rnk = 1
ORDER BY ranked.search_date;
```

## Answer / Result

Returns daily trending search queries.

## Insight

This query uses **window functions (RANK) with GROUP BY aggregation** to identify top-performing queries per day.

# 13. User Contribution to Total Ad Revenue

## Question

Calculate each user's percentage contribution to total revenue.

## SQL Query

```sql
SELECT a.user_id AS user_id,
ROUND(
100.0 * SUM(a.revenue) /
(SELECT SUM(revenue) FROM ad_clicks),
2
) AS revenue_percentage
FROM ad_clicks a
GROUP BY a.user_id
ORDER BY revenue_percentage DESC;
```

## Answer / Result

Returns revenue contribution percentage per user.

## Insight

This query uses **nested aggregation with GROUP BY normalization** to measure proportional revenue contribution.

# 14. Users with Above Average Watch Time

## Question

Find users whose average watch time is above platform average.

## SQL Query

```sql
SELECT v.user_id AS user_id
FROM video_views v
GROUP BY v.user_id
HAVING AVG(v.watch_time) >
(
SELECT AVG(watch_time)
FROM video_views
)
ORDER BY v.user_id;
```

## Answer / Result

Returns highly engaged video users.

## Insight

This query uses **HAVING clause with subquery comparison** to filter above-average user engagement.

# 15. Top 10 Users by Revenue with Search Activity

## Question

Find top 10 users by revenue with search activity and country.

## SQL Query

```sql
SELECT u.user_id AS user_id,
u.country AS country,
COUNT(DISTINCT s.search_date) AS search_days,
SUM(a.revenue) AS total_revenue
FROM users u
INNER JOIN search_queries s
ON u.user_id = s.user_id
INNER JOIN ad_clicks a
ON u.user_id = a.user_id
GROUP BY u.user_id, u.country
ORDER BY total_revenue DESC
LIMIT 10;
```

## Answer / Result

Returns top revenue-driving users with engagement metrics.

## Insight

This query combines **multi-table joins with GROUP BY aggregation** to correlate user engagement with monetization.

# 16. Most Watched Video Categories

## Question

Find most watched video categories.

## SQL Query

```sql
SELECT v.category AS category,
SUM(vw.watch_time) AS total_watch_time
FROM videos v
INNER JOIN video_views vw
ON v.video_id = vw.video_id
GROUP BY v.category
ORDER BY total_watch_time DESC;
```

## Answer / Result

Returns video categories ranked by engagement.

## Insight

This query uses **JOIN + SUM aggregation + GROUP BY ranking logic** to identify content consumption patterns.
